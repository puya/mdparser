# Package Name Suggestions

## Current Name
**mdparser** - Simple and clear, but quite generic

## Naming Considerations

The tool's key features:
- Extracts structured information from Markdown
- Optimized for AI-agent usage
- Section-limited search capabilities
- Processes large files efficiently
- Extracts headings, emphasized text, and sections

## PyPI Availability Check

**Note**: Most common names are already taken on PyPI. Here are some alternatives:

### Available Names (Need Verification)
- Names with `-cli` suffix (e.g., `mdextract-cli`)
- Names with creative variations (e.g., `mdxtract`, `mdxtractr`)
- More specific descriptive names

### Taken Names
- `mdparser` (taken, but old/inactive package from 2020)
- `mdsift`, `mdquery`, `mdextract`, `mdagent` - all taken
- Most common `md*` variations are taken

## Top Recommendations

### 1. `mdxtract` ⭐⭐⭐
**Why**: 
- Unique variation of "extract"
- Short and memorable (6 chars)
- Clear purpose
- Easy to type: `mdxtract document.md --headings 3`
- Professional yet distinctive

**Usage**: `mdxtract document.md --headings 3`

### 2. `mdxtractr` ⭐⭐
**Why**: 
- Even more unique with 'r' suffix
- Still clear purpose
- Very distinctive

**Usage**: `mdxtractr document.md --find "term"`

### 3. `mdextract-cli` ⭐⭐
**Why**: 
- Clear that it's a CLI tool
- Descriptive
- May be available on PyPI

**Usage**: `mdextract-cli document.md --headings 2`

### 4. `mdx` ⭐
**Why**: 
- Very short (3 chars)
- Modern feel
- Easy to type

**Usage**: `mdx document.md --headings 3 --format json`

### 5. Keep `mdparser` ⭐
**Why**: 
- Already established in your project
- Simple and clear
- Even though taken on PyPI, you can use it locally
- Generic enough for discoverability
- If publishing, could use namespace: `yourname-mdparser`

**Usage**: `mdparser document.md --headings 2`

## Considerations

### Length
- **Short (4-6 chars)**: `mdsift`, `mdquery`, `mdextract` - Easy to type
- **Medium (7-10 chars)**: `mdparser`, `mdstruct` - Balanced
- **Long (10+ chars)**: `markdown-extract` - Descriptive but verbose

### Clarity
- **Immediately clear**: `mdextract`, `mdsearch`
- **Requires context**: `mdsift`, `mdagent`
- **Generic**: `mdparser`, `mdinfo`

### Uniqueness
- **Unique**: `mdsift`, `mdagent`, `mdseek`
- **Common pattern**: `md-extract`, `markdown-extractor`
- **Generic**: `mdparser`, `mdinfo`

### CLI Usage
- **Easy to type**: Short names (`mdsift`, `mdquery`)
- **Professional**: Descriptive names (`mdextract`)
- **Memorable**: Unique names (`mdsift`, `mdagent`)

## Recommendation

**Top Choice: `mdsift`**
- Unique and memorable
- Implies sifting through content
- Short and easy to type
- Works for both extraction and search
- Professional yet distinctive

**Alternative: `mdquery`**
- Clear query/search functionality
- Short and professional
- Good for AI-agent use case

**Safe Choice: Keep `mdparser`**
- Already established
- Clear and simple
- Generic enough for discoverability
